# 提案：後台管理系統 (Admin Dashboard System)

## 變更 ID
`admin-dashboard-system`

## 目的 (Why)

NovaScribe 需要一個功能完整的後台管理系統,為內容創作者提供高效的文章管理、數據監控與 SEO 優化工具。後台系統是部落格平台的核心控制中心,直接影響創作者的工作效率與內容品質。

### 主要驅動因素
1. **內容管理效率**:創作者需要直觀的介面進行文章撰寫、編輯與發布
2. **數據洞察需求**:需要整合 GA 數據,提供閱讀統計與內容健康度分析
3. **SEO 優化支援**:為每篇文章提供獨立的 SEO 元數據編輯功能
4. **圖片資產管理**:本地圖片上傳與 AI 輔助的 Alt Text 生成
5. **版本控制**:類似 Git 的內容修改歷史追蹤
6. **讀者行為分析**:社群互動熱圖 (Heatmap) 功能

## 變更內容 (What)

本變更將建立完整的後台管理系統,包含以下核心模組:

### 1. 儀表板 (Dashboard)
- Google Analytics API 整合,顯示基本閱讀統計
- 內容健康度總覽 (關鍵字密度、標題結構、連結完整性)
- 最近文章與熱門文章快速檢視
- 系統狀態監控

### 2. 文章管理 (Article Management)
- Markdown 編輯器 (支援即時預覽與代碼高亮)
- 文章 CRUD 操作 (建立、讀取、更新、刪除)
- 發布狀態切換 (草稿/已發布/排程發布)
- 分類與標籤管理
- 內容版本控制與回溯

### 3. 圖片管理 (Image Management)
- 本地圖片上傳至指定資料夾
- 圖片路徑自動生成與引用
- AI 自動生成 Alt Text (無障礙與 SEO 優化)
- 圖片庫瀏覽與搜尋

### 4. SEO 設定 (SEO Configuration)
- 每篇文章的獨立 Meta Title 編輯
- Meta Description 編輯與 AI 自動生成建議
- Open Graph 圖片設定
- Canonical URL 與 Hreflang 配置
- 結構化數據 (JSON-LD) 預覽

### 5. 網站設定 (Site Settings)
- Google Analytics 追蹤碼配置
- Google Search Console 整合設定
- 網站基本資訊 (標題、描述、Logo)
- 主題與外觀設定

### 6. 社群互動熱圖 (Engagement Heatmap)
- 匿名追蹤讀者閱讀深度
- 停留點視覺化呈現
- 熱區分析報告

### 7. 權限管理 (Authentication & Authorization)
- 使用 Auth.js 進行身份驗證
- 角色管理 (Admin / Editor / Viewer)
- Session 管理

## 風險與挑戰

1. **GA API 整合複雜度**:需要處理 OAuth 驗證與資料同步
2. **Markdown 編輯器效能**:大型文章可能影響編輯器流暢度
3. **圖片儲存策略**:初期使用本地儲存,未來需遷移至 CDN
4. **AI Alt Text 生成品質**:需要調校 Prompt 確保輸出品質
5. **版本控制資料量**:需要設計合理的歷史保留策略
